Twitter API の消費

　Twitter Search は、ツイート検索の際に Twitter API を最大10個消費します。このため、1時間につき15回ほど検索可能です。ただし、日付指定を最新のツイートが生成された日より先に設定した場合、検索中に条件と合致したツイートが発見された場合、キャッシュ内で発見された場合、ユーザーが存在しない場合は1～10個の範囲で消費されます。

　現在、この検索システムが検索可能な範囲は最新から2000個までの過去のツイートです。それ以上の検索は可能ですが、 Twitter API の消費が著しくなり実用的でないため制限しています。一般的に Twitter が規制しているAPI数はIP判別で150回です。


Twitter REST API の使用

　ツイートデータは検索対象のユーザー・タイムラインから取得します。REST API は「GET statuses/user_timeline」を使用します。page パラメータで取得範囲を指定し、一回のリクエストで 200個のツイートデータをレシーブしています。レシーブしたデータはキャッシュに一時保存され、API数の消費を抑えます。なお、この REST API を使用して、保護されたユーザーのタイムラインを取得することはできません。

　REST API については Twitter Developers のドキュメンテーションを参照してください。

   Twitter Developers "GET statuses/user_timeline":
      https://dev.twitter.com/docs/api/1/get/statuses/user_timeline
